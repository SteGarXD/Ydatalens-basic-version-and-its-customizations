# Документация по кастомизациям DataLens

## Обзор

Все кастомизации для DataLens организованы в модульной структуре с использованием feature flags. Это позволяет легко включать/отключать функции и обновлять DataLens без потери изменений.

## Структура кастомизаций

### Брендинг (`branding/`)
- `logo.tsx` - Компонент логотипа
- `colors.ts` - Цветовая схема
- `fonts.ts` - Настройки шрифтов
- `theme.ts` - Объединенная тема

### Функции (`features/`)
Каждая функция в отдельной директории с собственными компонентами и логикой.

### Конфигурация (`config.ts`)
Централизованная конфигурация всех feature flags и настроек.

## Принципы кастомизации

1. **Изоляция** - все кастомизации в отдельных модулях
2. **Feature Flags** - условная загрузка через конфигурацию
3. **Патчи** - мелкие изменения через patch-package
4. **Модульность** - крупные изменения в отдельных модулях
